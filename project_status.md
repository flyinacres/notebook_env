?? Project Environment-Lock: Architecture & RoadmapCore PhilosophyA lightweight, self-contained Python snapshot tool for Jupyter notebooks. It prevents the "It worked on my machine" problem when distributing notebooks (especially for courseware/education) by embedding deterministic environment setup blocks into Cell 2 of .ipynb files.Key Architectural DecisionsGeneration Strategy (In-Environment Static AST):Must inspect the author's active working environment (importlib.metadata) where all packages/drivers are already installed and verified.Uses static ast parsing to target only explicit top-level imports, avoiding transitive package bloat.Strips hardware build tags (e.g., torch==2.3.0+cu121 $\rightarrow$ torch==2.3.0) so pip resolves local hardware binaries cleanly on recipient machines.Embedded directly in the notebook (Cell 2) so the notebook is a self-contained artifact requiring no adjacent files.Batch Processing Strategy:Operates as a fast, static linter/formatter directly against the active environment across a directory of .ipynb files.No headless nbconvert subprocess execution during generation—just pure AST parsing + metadata lookup.Validation / Smoke-Testing Strategy (Separate Harness):A standalone test script whose only job is CI/CD verification.Takes a generated notebook, creates a clean temporary .venv, runs pip install -r pinned_requirements.txt, and executes the notebook headlessly.No complex auto-patching magic: If it fails, it provides a clear diagnostic (e.g., "Missing scipy in Cell 4"),
